**Uncertainty quantification methods for probabilistic whole-building life cycle assessment: A comparative analysis**

Martín I. Torres · Wil V. Srubar III
University of Colorado Boulder

[![DOI](https://zenodo.org/badge/DOI/10.5281/zenodo.19226461.svg)](https://doi.org/10.5281/zenodo.19226461)

---

## Overview

This repository contains the data, source code, and analysis notebooks supporting the paper above. The study evaluates six combinations of probability estimation (PE) and weighting (WT) methods — referred to as PEWT methods — to determine which best represents the distribution of environmental carbon footprint data used in probabilistic whole-building life cycle assessment (pWBLCA).

The three PE methods compared are:

| Label | Method |
|---|---|
| **Normal** | Gaussian fit (mean and standard deviation) |
| **Lognormal** | Lognormal fit via weighted maximum likelihood estimation |
| **KDE** | Kernel density estimation with Scott's bandwidth rule |

Each PE method is evaluated under two WT strategies:

| Label | Strategy |
|---|---|
| **Uniform** | Equal weights across all data points |
| **Variable** | User-supplied weights that reflect heterogeneity in the dataset |

Performance is measured using Wasserstein distances between each fitted model and the empirical data distribution, and by comparing decision outcomes when the PEWT methods are applied in a pWBLCA calculation.

## Workflow

The analysis is organized into three sequential notebooks.

### Notebook 1 — Create data (`01_CompareUQ_CreateData.ipynb`)

Generates and preprocesses the datasets used in the analysis.

**Synthetic datasets.** Generates 15,000 synthetic datasets using `random_irregular_dataset()`, which creates multimodal distributions by mixing 1–5 randomly parameterized components (Normal, Skew-normal, Student-*t*, and Lognormal). Sample sizes range from 4 to 1,000 values per dataset (log-uniformly distributed). Each dataset is assigned random Dirichlet mixture weights.

**Empirical datasets.** Retrieves real embodied carbon coefficient (ECC) data by material type from the Embodied Carbon in Construction Calculator (EC3) database via API. Data are unit-normalized, and outliers are removed using the IQR method (Q1/Q3 ± 1.5 × IQR). Datasets are assigned random Dirichlet weights.

**Metadata computation.** Computes 20+ statistical metrics for each dataset using `empirical_metadata()`, including:
- Mean, coefficient of variation, skewness, excess kurtosis
- Shapiro–Wilk *W* statistic for Normal and Lognormal fits
- Mode count (estimated via KDE smoothing)
- Shannon entropy
- Outlier weight proportion
- Wasserstein-1 distance between weighted and unweighted distributions

Datasets with outlier metadata values (IQR ± 1.5 × IQR per metric) are flagged and excluded, and the synthetic set is trimmed to 10,000 datasets. Four-material combinations for pWBLCA are written to `combos.txt`.

**Key outputs:** `DATA_all.json`, `dct_realeccs_trimmed.json`, `datasets_outliers.json`, `datasets_trimto10k.json`, `combos.txt`

> **Note on raw EC3 data:** The empirical data pull from EC3 requires access to a local directory of pre-downloaded EC3 CSV files (`EPDsFromEC3/EPD_AllOfEC3/`), which is not included in this repository. This step is only needed to regenerate `dct_realeccs_trimmed.json` from scratch. The processed file is already included in `data/processed/`, so Notebooks 2 and 3 can be run without it.

---

### Notebook 2 — Analyze data (`02_CompareUQ_AnalyzeData.ipynb`)

Fits all six PEWT models to each dataset and evaluates their accuracy.

For each dataset, three PE models are fitted under both WT strategies, producing six PEWT fits. The accuracy of each fit is measured using:
- **Wasserstein-1 distance (W1):** Area between the empirical and model CDFs
- **Wasserstein-2 distance (W2):** L2 analog of W1, more sensitive to tail behavior
- **Kolmogorov–Smirnov statistic:** Maximum vertical distance between CDFs

PEWT methods are ranked by W1 distance for each dataset, and ranking frequencies are aggregated across datasets. Correlations between dataset metadata (from Notebook 1) and PEWT ranking are computed to identify which distributional characteristics favor or penalize each method.

**Key outputs:** `outputs/figures/FIG1_PDFandCDFofUQMethods.png`, `FIG2_Wass1DistStripAndRank.png`, `FIG3_WassRankFreq_Heatmap.png`, `FIG4_WassDistanceVsMetric.png`; summary tables in `outputs/tables/`

---

### Notebook 3 — Perform pLCA (`03_CompareUQ_PerformPLCA.ipynb`)

Applies the PEWT models to a simplified pWBLCA and measures how the choice of UQ method affects LCA decision outcomes.

For each four-material combination in `combos.txt`, Monte Carlo samples are drawn from each of the six PEWT fits and used to compute distributions of:
- Total embodied carbon intensity (ECI)
- Per-material contribution ranking (1st through 4th)
- EC reduction under a material-quantity reduction scenario
- EC reduction under an ECC cap scenario

Result metrics are aggregated by PEWT method to characterize how the choice of UQ method affects the spread and rank-ordering of pWBLCA outcomes.

**Key outputs:** `outputs/figures/FIG5_*`, `FIG6_PLCAVisualizeUQFits.png`, `FIG7_RanksByDatasetAndPEWT.png`

## Usage

Run the notebooks in order from the `notebooks/` directory:

```
01_CompareUQ_CreateData.ipynb   →   data/processed/
02_CompareUQ_AnalyzeData.ipynb  →   outputs/figures/, outputs/tables/
03_CompareUQ_PerformPLCA.ipynb  →   outputs/figures/, outputs/tables/
```

Notebooks 2 and 3 depend on the processed data files produced by Notebook 1. If you are reproducing the analysis from scratch, run them in sequence. Pre-generated data files are included in `data/processed/` so that Notebooks 2 and 3 can be run independently.
